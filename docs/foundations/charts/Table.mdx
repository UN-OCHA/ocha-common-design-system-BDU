# Table

Tables are data visualizations too — design them. Use when the reader needs precise values, or when many variables share a row.

## When to use

- Precise numeric lookup matters more than pattern at a glance.
- Many variables per row (country + sector + year + value + trend).
- 6–30 rows. Longer — add search/filter.
- Avoid for ranking one variable — a horizontal bar reads faster.
- Avoid left-aligning numeric columns — they must right-align.

## Apply it in your tool

### Power BI
- Visual: **Table** (not matrix unless pivoting).
- Header: background `#009EDB`, font color `#FFFFFF`, Roboto Condensed Bold 11pt.
- Rows: alternating `#F2F2F2`; text `#333333`, Roboto Condensed 11pt.
- Numeric columns: set **alignment = right**, format with `k/M/B` prefix.

### Datawrapper
- Chart type: **Table**.
- Custom theme OCHA. Right-align numeric columns in *Visualize*.

### Chart.js / D3
Use native HTML `<table>`:
```css
thead th { background: #009EDB; color: #fff; font: 700 11px/2 'Roboto Condensed'; padding: 0 8px; text-align: left; }
tbody tr:nth-child(even) td { background: #F2F2F2; }
tbody td { font: 400 11px/2 'Roboto Condensed'; color: #333; padding: 0 8px; }
tbody td.num { text-align: right; font-variant-numeric: tabular-nums; }
```

### Illustrator
Use the [Humanitarian Dataviz Tool](https://brand.unocha.org) — it applies all these values automatically.
