# Vertical bar

Compares values across short, discrete categories or time increments (years, months, quarters).

## When to use

- Time series where each increment is discrete (annual, quarterly, monthly).
- Short category labels that read horizontally.
- 3–12 bars. More than that — switch to horizontal or a line chart.
- Avoid rotating X-axis labels. If labels don't fit horizontally, switch to a horizontal bar.
- Avoid when values vary by orders of magnitude — a log-scale bar confuses readers.

## Apply it in your tool

### Power BI
- **Format → Columns → Fill:** `#009EDB`. **Corner radius:** `0`.
- **Data labels:** On, Position **Outside end**, Roboto Condensed 10pt.
- **Y axis:** Start at `0`. **Gridlines:** Off (or `#E6E6E6` at 0.5pt if critical).
- **X axis → Values → Layout:** Horizontal only.

### Datawrapper
- Theme: **OCHA**.
- Manual: *Appearance → Bars → Color* `#009EDB`; *Axis* → baseline Y = 0; labels horizontal.
- If categories don't fit horizontally, switch chart type to **bar** (horizontal).

### Chart.js / D3
```js
new Chart(ctx, {
  type: 'bar',
  data: { labels, datasets: [{ data, backgroundColor: '#009EDB', borderRadius: 0 }] },
  options: {
    plugins: { legend: { display: false } },
    scales: {
      x: { grid: { display: false }, ticks: { font: { family: 'Roboto Condensed', size: 11 }, color: '#555555', maxRotation: 0 } },
      y: { beginAtZero: true, grid: { color: '#E6E6E6', drawBorder: false }, ticks: { font: { family: 'Roboto Condensed', size: 10 } } }
    }
  }
});
```
