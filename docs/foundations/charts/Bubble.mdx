# Bubble (surface area)

Communicates relative magnitude. The eye compares area, not precise value — so use it for communication pieces, not analysis.

## When to use

- A communication piece where relative size is the message (e.g. "these crises are bigger").
- 3–8 bubbles. More becomes a cloud.
- Values span roughly 1 order of magnitude (10×). Beyond that, small bubbles disappear.
- Avoid for analytical dashboards — readers can't compare areas accurately. Use a horizontal bar instead.
- Avoid matching bubble size to **diameter** — always scale by **area** (`area = π × r²`).

## Apply it in your tool

### Power BI
- Scatter chart → turn off X and Y axes → size by value.
- **Fill:** `#009EDB`. **Outline:** none. **Size range:** min 6px, max 40px (relative to canvas).
- Ensure Size property is mapped to `value`, not `count of value` — otherwise it scales by diameter.

### Datawrapper
- Use the **Symbol map** or a **scatter** chart. Size bubbles by area. Fill `#009EDB`.

### Chart.js / D3
```js
// D3: scale by area
const r = d => Math.sqrt((d.value - min) / (max - min) * (maxR*maxR - minR*minR) + minR*minR);
svg.selectAll('circle').data(data).join('circle')
  .attr('r', r).attr('fill', '#009EDB');
```

### Illustrator
Use the [Humanitarian Dataviz Tool](https://brand.unocha.org) — it applies all these values automatically.
