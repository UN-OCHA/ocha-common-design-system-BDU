# Pie

Use sparingly — only when proportion is the single message and slices are 2–3. Readers compare angles poorly; a bar chart almost always beats a pie.

## When to use

- 2–3 slices. Never more than 5.
- Proportion is the only message — not absolute value, not rank.
- Single pie, not comparisons side-by-side.
- Avoid 3D, rainbow colour per slice, exploded slices.
- If the reader needs to compare sizes, use a horizontal bar.

## Apply it in your tool

### Power BI
- Visual: **Pie chart**. Start angle `0`. Clockwise.
- Series colors: OCHA ramp (`#009EDB`, `#0074B7`, `#64BDEA`).
- Outline: `#FFFFFF`, width `2`.
- Data labels: Percentage, Roboto Condensed bold, inside for large slices.

### Datawrapper
- Chart type: **Pie**. Outline: white 2px.
- Labels: % only; inside dark slices in white.

### Chart.js / D3
```js
new Chart(ctx, {
  type: 'pie',
  data: { labels, datasets: [{ data, backgroundColor: ['#009EDB', '#0074B7', '#64BDEA'], borderColor: '#FFFFFF', borderWidth: 2 }] },
  options: { plugins: { legend: { position: 'bottom', labels: { font: { family: 'Roboto Condensed', size: 11 } } } } }
});
```
