# Sankey

Flow diagram showing how quantities move between source and target nodes. Use for funding flows, displacement routes, supply chains.

## When to use

- Flow data: source → target → (optional) next level, with a value on each link.
- You want the reader to see magnitudes **and** connections at once.
- 3–30 nodes. Beyond that, splits become illegible.
- Avoid when the story is a ranking — use a horizontal bar.
- Avoid when flows cross too many times — re-order nodes to minimise crossings.

## Style values

| Element | Value | Token |
| --- | --- | --- |
| Default node color | `#009EDB` | `--brand-primary` |
| Multi-level node ramp | `#009EDB`, `#0074B7`, `#64BDEA` | OCHA ramp |
| Node width | `20px` | `sankeyNodeWidth` |
| Node padding | `15px` | `sankeyNodePadding` |
| Link opacity | `0.4` | `sankeyLinkOpacity` |
| Link fill | source-node color | — |
| Label font | Roboto Condensed 11px, `#555555` | — |
| Label mode | both ends default | `sankeyLabelMode: 'both'` |

## Apply it in your tool

### Power BI
- Install **Sankey** custom visual (from AppSource).
- Node colors: OCHA ramp by level. Node width 20px. Link opacity ~40%.
- Format labels to Roboto Condensed 11pt.

### Datawrapper
- Chart type: **Sankey diagram** (Datawrapper Plus).
- Colors: custom, OCHA ramp in source order.

### Chart.js / D3
```js
// D3 sankey
const sankeyLayout = d3.sankey().nodeWidth(20).nodePadding(15).extent([[0,0],[w,h]]);
const { nodes, links } = sankeyLayout({ nodes: n.map(d=>({...d})), links: l.map(d=>({...d})) });
svg.selectAll('rect').data(nodes).join('rect')
  .attr('fill', d => ['#009EDB','#0074B7','#64BDEA'][d.depth] || '#009EDB');
svg.selectAll('path').data(links).join('path').attr('stroke', d => d.source.color).attr('stroke-opacity', 0.4);
```
