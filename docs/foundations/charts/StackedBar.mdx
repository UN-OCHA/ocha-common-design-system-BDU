# Stacked bar

Shows totals broken into 2–3 meaningful sub-categories. Reader compares totals *and* composition in one view.

## When to use

- Each total has a clear 2–3 category breakdown (funded/unfunded; reached/unreached; sector mix).
- The category order is consistent across all bars.
- Rows are sortable by total.
- Avoid more than 3 segments per bar — readers can no longer compare sub-segments by length.
- Avoid when the composition is the *only* message — use a 100% stacked instead.

## Style values

| Element | Value | Token |
| --- | --- | --- |
| Primary segment | `#009EDB` | `--brand-primary` |
| Secondary segments (OCHA ramp) | `#C5DFEF`, `#64BDEA`, `#0074B7` | `--cd-blue--brighter`, `--cd-blue--bright`, `--cd-blue--darker` |
| Segment separator stroke | `#FFFFFF`, 1px | — |
| Bar thickness (md) | `24px` | — |
| Bar gap | `10px` | — |
| In-segment value (only if segW > 24px) | Roboto Condensed 9px, contrast-text auto | — |
| Total label | Roboto Condensed 10px, `#333333`, 6px outside bar | — |
| Max segments | `3` before legibility drops | — |

## Apply it in your tool

### Power BI
- **Stacked bar chart** visual. Bar corner radius `0`.
- Series colors: `#009EDB`, `#C5DFEF` (OCHA ramp in descending order).
- Data labels: inside segment only when segW > ~24px; always show total.
- Sort by total descending.

### Datawrapper
- Theme: **OCHA**.
- Manual: *Appearance → Stacked bars*, custom series colors from OCHA ramp; enable **totals** on.

### Chart.js / D3
```js
new Chart(ctx, {
  type: 'bar',
  data: { labels, datasets: [
    { label: 'Funded', data: funded, backgroundColor: '#009EDB' },
    { label: 'Unfunded', data: gap, backgroundColor: '#C5DFEF' }
  ]},
  options: { indexAxis: 'y', scales: { x: { stacked: true, beginAtZero: true }, y: { stacked: true } } }
});
```

Use the [Humanitarian Dataviz Tool](https://brand.unocha.org) — it applies all these values automatically.
