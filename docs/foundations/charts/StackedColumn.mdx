# 100% stacked (stacked column)

Shows composition only — every bar totals 100%. Use when the mix is the message and the absolute value is not.

## When to use

- Comparing composition across time periods or categories.
- Each bar must sum to 100% — values are normalized.
- 2–3 sub-categories work best; 4 is the hard ceiling.
- Avoid when the reader needs the absolute total — use a stacked bar with totals instead.
- Avoid when one slice dominates so heavily that the others are unreadable.

## Apply it in your tool

### Power BI
- Use **100% Stacked column** visual. Corner radius `0`.
- Normalize data in Power Query (`value / sum(value) over (bar)`) or rely on visual's 100% mode.
- Series colors from the OCHA ramp in sequence.

### Datawrapper
- Chart type: **Stacked column**, then toggle **Show as 100%**.
- Labels: only percentage; hide Y axis.

### Chart.js / D3
```js
new Chart(ctx, {
  type: 'bar',
  data: { labels, datasets },
  options: {
    scales: {
      x: { stacked: true },
      y: { stacked: true, max: 100, ticks: { callback: v => v + '%' } }
    }
  }
});
```

### Illustrator
Use the [Humanitarian Dataviz Tool](https://brand.unocha.org) — it applies all these values automatically.
